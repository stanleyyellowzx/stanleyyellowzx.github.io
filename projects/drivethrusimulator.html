DriveThru Simulator

Last updated: May 15 2025

Project Date: March 2024

Technologies used: C

I created this project for a systems programming, and it is also the first time I created a project using C. The program simulates a drive-thru of a restaurant by using concurrent programming techniques involving signals, UDP sockets, and multi-threading with semaphores.


This image here is what the drive-thru simulation looks like. This graphical interface was created using C's x11 library, although I was not the one who created this graphical interface. I created all of the logic in the backend to have customers show up at the drive-thru. The simulation has a generator which creates a bunch of customer processes, which then connects to a restaurant server. Then, there is then an order taker to take the orders of the customers. To handle customer lineups, I used semaphores to make sure the customers were in the correct place in line.


Reflections

This was the first time that I had to work with concurrent programming. At the time that I created this project, I think that this was definitely the most challenging project I created, since concurrent programming was a new concept to me. I think the difficulty comes with the fact that you have multiple processes running at the same time, and they could be sharing resources between each other, so you need to find a way to coordinate the processes so that multiple processes don't try and access the same resource at the same time.
