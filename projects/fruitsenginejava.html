Fruits Search Engine (Java Version)

Last updated: May 14 2025

Project Date: December 2023

Technologies used: Java, JavaFX

Previously, I implemented a fruits search engine project using Python, and here I've reimplemented it using Java. With Java, I was able to create a GUI using JavaFX to allow users to be able to perform search queries through an interface rather than a terminal. This project scrapes data from sites with a format like this link: Link and then calculates TF-IDFs for all of the unique words that are found and page ranks which are used to determine which pages show up for search queries. Depending on the amount of incoming/outgoing links, the scraping can take anywhere from a few seconds to 2 minutes.


This here is what the GUI looks like. Users are able to enter their search query in the long bar, the number of results they would like to see in the box labeled "enter number of searches" and then the user can add a boost to their search. The boost uses an algorithm to return results that are more relevant to the search query instead of just pages that spam a particular word to try and get search queries to match to those pages.


Reflections

Implementing this version of the fruits search engine was much easier than the Python version since I already had experience with Java. I also feel that Java was a better tool for this task as it involved scraping and parsing data, and then saving that data into files. I would say that the scraping and parsing data was simpler with Python, but saving calculated values such as TF-IDFs and page ranks was much simpler as you are able to save entire objects to files.
